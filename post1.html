A Pixel is Never Just a Pixel

In solar radio astronomy, a pixel on your screen isn't just a dot of color; it represents a strict physical block of time (Δt) and frequency (ν). Currently, radiospectra maps these properties using disconnected NumPy arrays. This leads to a disjointed data structure where adding advanced features requires writing custom, scratch-pad math.

The Shift to NDCube & ExtraCoords

The biggest architectural shift I am implementing is replacing the custom GenericSpectrogram class with a coordinate-aware Spectra object backed entirely by ndcube.NDCube.

Why NDCube? Because it inherently understands Astropy's APE-14 World Coordinate System (WCS). For standard linear spectrograms, the mapping from pixel coordinates p to physical WCS values x is defined by the affine geometric transformation:

xi = Σ Cij (pj - p0,j) + x0,i

However, many solar radio instruments exhibit non-linear scalar frequency dispersions. To resolve this, I will utilize ndcube.ExtraCoords, storing irregular 1D frequency arrays as strict lookup tables. This natively computes physical coordinates without assuming linearity!

Cutting Through the Noise: Background Subtraction

Radio spectra are fundamentally contaminated by steady background emissions and systemic instrument drift. I am building a class-based background reduction architecture boasting O(N) vectorizations.

For example, implementing a temporal rolling minimum background subtraction with a window size W looks like this:

S'f, t = Sf, t - min(Sf, k)
for k ∈ [t - W/2, t + W/2]

This will be exclusively implemented utilizing scipy.ndimage.minimum_filter1d to enforce C-level vectorization and explicitly avoid Python bottleneck loops.

Minding the Gaps

Instrument telemetry is rarely perfect. Continuing the work I started in PR #181, I am building a standalone gap detection utility. It will measure strict differential deltas. Given an expected cadence C and a tolerance ε, the system will recognize a physical gap when:

Δti > C · (1 + ε)

When identified, the arrays will be patched with explicit np.nan blocks. This instructs Matplotlib to render clean, accurate visual whitespace instead of misleadingly stretching the data.
